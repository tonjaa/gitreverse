# gitreverse — Summary

> **Reverse-engineer any public GitHub repository into a single "vibe coding" prompt.**

---

## What It Does

`gitreverse` takes any public GitHub repository URL and transforms it into one short, conversational prompt — the kind you would paste into **Cursor**, **Claude Code**, **Codex**, or any other AI coding assistant to recreate the project from scratch.

---

## How It Works

1. **Input** — Paste a public GitHub repo URL or `owner/repo` slug on the home page.
2. **Fetch** — The app pulls the repo's metadata, a root file tree (depth 1), and the README.
3. **Generate** — An LLM (via OpenRouter) produces one concise, plain-language "vibe coding" prompt grounded in that context.
4. **Output** — Copy the prompt and paste it into your AI coding tool of choice.

> 💡 **Shortcut:** Replace `hub` with `reverse` in any GitHub URL to instantly run the flow.
> e.g. `github.com/vercel/next.js` → `gitreverse.com/vercel/next.js`

---

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Next.js (App Router) |
| UI | React + TypeScript + Tailwind CSS |
| Data Source | GitHub API |
| LLM Backend | OpenRouter (default: `google/gemini-2.5-pro`) |
| Caching (optional) | Supabase |

## Setup & Installation

```bash
# 1. Clone the repo
git clone https://github.com/filiksyos/gitreverse.git
cd gitreverse

# 2. Install dependencies
pnpm install

# 3. Configure environment variables
cp .env.example .env.local
# Required: OPENROUTER_API_KEY
# Optional: OPENROUTER_MODEL, GITHUB_TOKEN, Supabase vars

# 4. Run locally
pnpm dev
```

---

## Environment Variables

| Variable | Required | Description |
|---|---|---|
| `OPENROUTER_API_KEY` | ✅ Yes | Your OpenRouter API key |
| `OPENROUTER_MODEL` | ❌ Optional | Override the default LLM model |
| `GITHUB_TOKEN` | ❌ Optional | Increases GitHub API rate limits |
| Supabase vars | ❌ Optional | Enables server-side response caching |
